**Digest auth — FIX from sip_open_door.py:**
Replace `make_digest_response(username, password, realm, nonce, method, uri)` with HA1-mode:
```python
def make_digest_response(ha1: str, nonce: str, method: str, uri: str) -> str:
    ha2 = md5(f"{method}:{uri}")
    return md5(f"{ha1}:{nonce}:{ha2}")
```
This is the critical fix — Asterisk uses md5secret, not cleartext password.

**6 approaches (from RESEARCH.md investigation queue):**

Approach 1 — MESSAGE to Asterisk port 5060, TO=00e00000, body=OPEN_DOOR_IND (text/plain), digest auth with HA1.

Approach 3 — MESSAGE to Asterisk port 5060, TO=00e00000, body=JSON `{"method":"OPEN_DOOR_IND","from":"<sip:00700100002@192.168.1.88>","callid":0,"srcaddr":"00e70002","dstaddr":"00e00000"}` with content-type application/json. Also try TO=00800000000 with same JSON body.

Approach 4 — MESSAGE to Asterisk port 5060, TO=00401200000 (Kamailio trunk user), body=OPEN_DOOR_IND (text/plain), digest auth with HA1. Tests whether Kamailio trunk user triggers a different dialplan.
